# 🏫 AIZS｜校园智能咨询平台 - 答辩演示脚本

本脚本为项目答辩或汇报演示提供标准化流程指引，包含**准备工作**、**演示序列**以及**常见问题（FAQ）备答提示**，帮助演示者流利展示系统亮点。

## 🛠️ 1. 演示准备工作

在演示开始前 5 分钟，请确保完成以下准备：
1. **环境检查**：
   - 确认 `.env` 中已正确配置 `DASHSCOPE_API_KEY`。
   - 确认网络连接正常（百炼大模型 API 需要公网访问）。
2. **数据初始化**（可选）：
   - 若要展示纯净系统的导入流程，可先执行数据重置脚本：
     ```bash
     python scripts/reset_demo_data.py --yes
     ```
   - 提示：重置后数据库将清空，您可以现场演示“知识库批量导入”过程。
3. **服务启动**：
   - 启动统一门户中心：
     ```bash
     streamlit run app.py
     ```
   - 系统会自动在默认浏览器中打开统一分流门户主页 `http://localhost:8501`。
   - 首页为产品级门户页面，顶部展示品牌区，中部展示用户端和管理端两个入口卡片，底部展示功能亮点。
   - *(注：旧版 user_app.py 和 admin_app.py 已归档，答辩演示请统一启动 app.py)*
4. **演示资料就绪**：
   - 将演示所用的 PDF/Word/ZIP 压缩包文档放置在易于查找的位置（例如项目自带的 `demo_documents/` 文件夹）。

## 🧭 2. 核心演示序列（5分钟黄金答辩路线）

建议按照以下模块和顺序进行现场演示：

### 模块一：统一门户与多端分流展示（1分钟）
1. **控制大屏展示**：向评委展示 `app.py` 启动的统一门户界面，介绍“用户端”与“管理端”的核心功能划分与安全性边界。首页包含品牌区、功能标签、两个入口卡片和功能亮点展示。
2. **隐藏侧边栏演示**：向评委说明门户首页通过配置 `.streamlit/config.toml` 和 CSS 注入，完全隐藏了 Streamlit 默认的侧边栏导航，提供了更加专业和沉浸的页面排版。全局 CSS 通过 `utils/ui_utils.py` 统一管理，确保各页面风格一致。

### 模块二：管理后台与知识库导入与管理（1.5分钟）
1. **登录控制后台**：点击首页“进入管理端”按钮。系统将自动切换到管理员登录表单。输入账号：`admin`，密码：`admin123` 进行登录。
2. **演示统一知识库入口**：
   - 登录成功后，在侧边栏点击 **📚 知识库导入与管理**。
   - 页面分为三个 Tab：「📤 导入知识库」、「📂 已入库文档」、「🧹 清理失败记录」。
   - 点击“**一键导入演示知识库**”（或者演示拖拽多文件及上传 ZIP 压缩包）。
   - 观察页面的进度条加载状态，导入完成后，展示详细的处理结果列表与汇总指标卡片。
   - 点击“📥 下载本次导入报告（CSV）”，向评委展示生成的标准化审计报告。
   - **演示知识库批量管理与清洗**：切换到“📂 已入库文档” Tab，向评委展示使用高性能 `st.data_editor` 表格展示的文档列表。点击“☑️ 全选当前列表”按钮，或勾选部分文档，点击“⬜ 取消全选”可取消。然后点击“🗑️ 批量删除已选文档”，现场演示二次安全警告和物理文件拦截保护。向评委指出该操作与“🧹 清理失败记录” Tab 为完全隔离的独立功能，分工明确。

### 模块三：用户端精准问答与 Agent 多步推理（1.5分钟）
1. **切换至用户端**：点击管理端侧边栏的“🏠 返回系统首页”按钮，回到首页后点击“进入用户端”。
2. **基于知识问答与溯源**：
   - 输入：`“自习室怎么预约？有次数限制吗？”`。
   - 展示：渲染精准的回答，并展开折叠面板“查看知识库参考来源”，向评委展示系统如何精准追溯至源文件名称、物理页码（如：PDF 第 2 页）、匹配的余弦距离以及参考文本切片。这证明了在演示知识库覆盖范围内可稳定召回，杜绝幻觉。
3. **Agent 执行轨迹展示**：
   - 点击侧边栏 **🤖 Agent 智能演示** 导航菜单。
   - 输入：`“新生报到要准备什么？明天南京天气怎么样？”`
   - 观察页面实时渲染的“**Agent 执行轨迹**”流程步骤（包括分类为【招生】、调用检索工具、调用天气工具、合并结果输出），体现出 Agent 能够将复合问题拆解为多步求解。工具调用结果以卡片形式展示，失败调用显示中文错误提示。

### 模块四：运营看板与自检闭环（1分钟）
1. **数据看板审计**：
   - 点击侧边栏的“🏠 返回系统首页”，并重新点击“进入管理端”（此时自动处于已登录状态）。
   - 选择 **📊 数据看板**，浏览实时问答趋势、用户提问意图分布、待人工优化低质量问答清单，并着重向评委展示底部的“**知识库未命中问题 Top 10**”排行榜，说明它是如何实现数据自闭环运营的。空数据时页面会显示友好的中文提示，不会显示空白图或报错。
2. **系统一键自检**：
   - 进入 **🩺 系统自检** 页面。
   - 运行“一键健康检查”，向评委展示 12 项包含根目录校验、ChromaDB 集合状态、阿里云百炼 API 双模型（Qwen-Plus、Text-Embedding-V3）连通性及耗时测试的自检报告，体现系统的企业级健壮性。API Key 不会明文展示，模型连通性测试需按钮手动触发。未配置 API Key 时页面不会崩溃。

## 💬 3. 常见问题（FAQ）备答提示

在答辩问答环节，评委可能提出以下问题，建议按下述要点回答：

### Q1: 你们的系统是如何解决大模型“幻觉”问题的？
* **回答要点**：
  - 本系统采用了 **RAG（检索增强生成）架构**。在预置演示知识库覆盖的范围内，系统可以稳定召回。
  - 当学生提问时，系统不是直接让大模型作答，而是先将问题转化为向量，在 Chroma 本地向量数据库中进行相似度检索，召回最相关的校园规章制度切片。
  - 然后，系统将这些召回的“知识切片”作为上下文，配合精心设计的 System Prompt 约束，指示大模型“只能根据参考资料回答，如果资料中没有，请委婉拒绝”。
  - 同时，我们在前端为每次回答都保留了“知识来源明细”，可追溯至具体文件和页码，确保了回答的真实性和可信度。

### Q2: 意图路由（Intent Routing）是如何实现的？为什么不直接把所有问题丢给向量库？
* **回答要点**：
  - 我们实现了**双路意图路由机制**：首先通过规则匹配进行极速响应，其次使用 LLM 进行意图识别分类。
  - 在升级后的 RAG 架构中，分类不再作为硬限制阻断检索，而是采用“全库检索优先 + 分类增强 + 通用资料补充”的多路召回重排策略。意图分类仅作为检索加权的 boost 因子，这既在排序上突出了相关领域的资料，又保证了在分类不准或资料跨类时系统仍然能够从全库召回真实相关的内容。

### Q3: 你们为什么要把用户端和管理端整合在同一个 app.py 中？
* **回答要点**：
  - 在 Streamlit 多页面单体应用架构中，使用统一入口启动（`streamlit run app.py`）能大大简化部署和使用体验，无需管理员在多个端口分别启动不同的 Python 进程。
  - 我们通过 Streamlit 的 **st.session_state 状态机** 控制当前的运行模式（Home 首页、User 用户端、Admin 管理后台），并在同个进程内分发渲染不同的视图，保障了良好的内存共享和一致的会话状态。
  - 同时，统一入口在 `app.py` 中完成侧边栏导航隐藏、状态机分流和管理员登录拦截。在未成功登录前，系统不会加载任何管理端子页面，保证了系统前台的安全性与轻量性。
  - 全局 CSS 样式通过 `utils/ui_utils.py` 统一管理，所有页面调用 `inject_global_css()` 注入样式，确保视觉风格一致。空值保护通过 `utils/display_utils.py` 统一处理，避免空值参与运算导致页面崩溃。
