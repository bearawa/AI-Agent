# 🏫 AIZS｜校园智能咨询平台

面向学生与家长的校园智能咨询系统 —— 集 RAG 检索增强生成、意图路由、Agent 多步推理、工具调用与回答质量评估于一体。

本项目是一个全功能的校园智能客服平台，覆盖 **基础档**、**进阶档** 和 **挑战档** 三个层级的完整功能实现。系统采用基于 **Streamlit** 的多页面单体应用架构，内部按页面层、服务层、数据访问层和存储层进行分层设计。系统采用 **Python 3.11+**、**ChromaDB**、**SQLite** 构建数据存储，接入阿里云百炼 **qwen-plus** 大语言模型与 **text-embedding-v3** 向量模型，全程纯中文环境开发。

## 🌟 项目亮点

1. **双路意图路由与全库检索优化**：实现了“规则优先 + 大语言模型兜底”的意图分类引擎，准确识别招生、学务、后勤、校园生活四大意图。系统采用“全库检索优先 + 分类增强 + 通用资料补充 + 自动回退”检索策略，意图与文档分类仅作为排序辅助加权，绝不作为限制检索的硬条件，有效避免了分类偏差导致相关内容被过滤的问题。
2. **知识库统一导入与管理中心**：将单文件导入、批量导入、知识库管理合并为统一的「📚 知识库导入与管理」页面。支持**上传单文件或多文件**（上传一个文件即为单文件导入，上传多个即为批量导入）、**ZIP 压缩包导入（含 Zip Slip 路径穿越防御）**、**演示知识库一键导入**。新增**全选/取消全选**、**批量勾选删除（含二次确认）**和**一键清理失败记录**功能。
3. **企业级级联删除**：在管理后台提供强一致性数据级联清理服务，物理删除文档的同时，同步清理向量库切片、数据库索引以及所有被引用的消息片段。
4. **主动澄清与 Function Calling 复合推理**：支持对模糊校园问题的拦截询问；内置多步推理 Agent，能够将复合提问拆解并协调校历、天气及知识库多渠道工具联合求解。
5. **全生命周期质量审计**：内置规则与 LLM 双重回答质量自动评估系统，即时对生成的每次应答进行评分打标签，方便管理运营。
6. **数据驱动的未命中监控**：后台提供专项“无来源提问 Top 10”及“无来源率”统计，使管理人员清晰捕捉学生最关心但本地知识库尚未收录的政策盲区。
7. **一键式系统自检**：提供涵盖核心文件、数据库连接、向量库就绪状态及第三方服务可用性的 12 项系统自检功能。

## 🎯 功能总览

### 一、基础档功能
- **多格式文档解析**：支持 `.pdf`、`.docx`、`.txt` 格式的规章制度文件解析，内置滑动窗口与重叠策略中文语义切分器。
- **物理页码/片段自适应溯源**：PDF 解析保留物理页码信息；Word/TXT 自动生成切片片段编号，确保引用出处真实可查。
- **RAG 向量检索与阈值过滤**：使用 ChromaDB 进行本地持久化检索，在演示知识库覆盖范围内可稳定召回符合余弦距离阈值（默认 0.8）的语义内容。
- **多轮改写追问**：自动提取历史上下文，通过大模型重写当前问题，消除代词和上下文信息缺失。
- **流式对话与引用展示**：支持流式文字渲染，回答下方可展开查看原始知识库来源切片明细。

### 二、进阶档功能
- **管理后台 — 知识库导入与管理中心**：将单文件导入、批量导入、知识库管理整合为统一的「📚 知识库导入与管理」入口，支持：
  - **多格式文档导入**：上传一个或多个 PDF/DOCX/TXT；上传单个文件即等同于原单文件导入
  - **ZIP 压缩包导入**：自动解压并过滤格式，内置 Zip Slip 路径穿越攻击防御
  - **演示文档一键导入**：从 demo_documents/ 目录批量入库，SHA-256 去重自动跳过
  - **已入库文档批量管理（全新升级）**：采用高性能表格展示，支持在表格中多选、一键"全选当前列表"、"取消全选"文档，提供统一批量删除按钮，并设有二次确认安全防御机制，级联清理向量切片、数据库索引与磁盘物理副本（安全保护 `demo_documents` 物理免删）。
  - **独立的失败记录清理**：与手动勾选批量删除完全独立，专门用于一键清理由于导入解析失败而残留的 failed/error 等无效状态记录，互不干扰。清理前需二次确认，明确说明该操作只清理失败导入记录，不会删除已成功入库的知识库文档。
  - **导入结果 CSV 下载**：记录每个文件的状态、分类、切片数与错误原因
- **意图路由与检索加权**：支持通用资料、招生、学务、后勤、校园生活、其他六大意图分类。检索时基于“全库优先+分类增强+通用资料补充”的 retrieve 机制，自动对各分类相似度分数进行轻量排序加权，既实现相关度加权增强，又避免了分类硬过滤阻断真实资料的召回。
- **数据看板与未命中统计**：实时运营卡片，包括问答趋势、意图分布。特别支持 **知识库未命中统计**（无来源提问 Top 10，无来源回复统计与无来源率走势）。
- **审计日志**：保留详尽的会话日志，提供多维筛选（含只看低满意度与无来源回复筛选），支持折叠展开以追溯每次交互耗时、改写词和参考片段。
- **点赞点踩反馈**：每个回答均具备 👍/👎 评分，可填写优化意见，实时联动数据看板。

### 三、挑战档功能
- **Agent 推理引擎**：基于 Function Calling 的自主多步推理，支持模糊问题主动拦截澄清。
- **多工具注册调用**：包含天气查询 (`get_weather`)、校历查询 (`get_school_calendar`)、校园服务状态 (`get_campus_service_status`) 与知识库检索 (`search_campus_knowledge`)。
- **Agent 执行轨迹可视化**：前端动态绘制 Agent 的执行轨迹图（意图分析 → 意图分类 → 工具决策 → 调用执行 → 结果归纳），并持久化存储运行轨迹记录。
- **回答质量评估**：双重评估器（规则加减分与 LLM 多维审查），自动标记 1-5 分和低质量问答，可在“回答质量评估”管理页面中查看分析。

## 🖥️ 双端分离设计与启动入口

为贴合真实的生产环境架构，AIZS 实现了**双端页面与入口分离**。系统通过唯一入口命令 `streamlit run app.py` 启动，内部采用状态机进行分流控制，普通咨询用户和管理员通过首页两个入口卡片进入各自的系统：

### 🧭 统一入口与多端分流
系统仅通过唯一的入口命令启动：
```bash
streamlit run app.py
```

在运行过程中，系统采用状态机（App Mode State）进行端状态的判定与分流控制：

1. **统一门户首页 (Home 模式)**：
   - 系统的引导首页，产品级门户页面，通过 `.streamlit/config.toml` 配置与 CSS 注入，**完全隐藏默认侧边栏**。
   - 顶部品牌区展示平台名称、简介和功能标签；中部两个入口卡片（用户端/管理端）；底部功能亮点展示。
2. **用户咨询端 (User 模式)**：
   - 面向学生与家长。点击"进入用户端"按钮一键进入。
   - 侧边栏包含：`💬 智能咨询`、`⏳ 对话历史`、`🤖 Agent 智能演示` 和 `🏠 返回系统首页`。
   - 统一 CSS 风格，聊天气泡、来源引用卡片、空数据友好提示。
   - 杜绝展示任何管理页面与配置入口。
3. **管理后台 (Admin 模式)**：
   - 面向管理员与老师。点击"进入管理端"按钮一键进入。
   - **安全校验**：首次进入仅显示管理员登录框，登录框居中展示。不加载任何管理端子页面，不访问数据库，不连接 ChromaDB 向量库，保证系统启动的安全与轻量。
   - 登录成功后，侧边栏展示：`📚 知识库导入与管理`、`📊 数据看板`、`📜 会话日志`、`🔌 工具调用日志`、`🎯 回答质量评估`、`🩺 系统自检` 以及 `🚪 退出登录` 与 `🏠 返回系统首页` 按钮。

*(注：旧版 `user_app.py` 和 `admin_app.py` 已归档至 `archive/legacy_entrypoints/`，启动请统一使用 `streamlit run app.py`。)*

## ⚙️ 环境配置与安装

### 1. 物理目录更名说明（重要）

若当前项目工作目录需修改，请在交付前手动重命名目录名称以满足 AIZS 的标识规范：
1. **停止运行**：按下 `Ctrl+C` 停止当前所有 Streamlit 运行实例。
2. **关闭 IDE**：关闭所有正在访问该目录的编辑器和命令行终端。
3. **重命名**：在资源管理器中将文件夹重命名为 `AIZS`。
4. **重新打开**：使用编辑器重新打开重命名后的 `AIZS` 工作目录。

### 2. 虚拟环境（.venv）的处理

由于物理路径发生了改变，如果您之前在项目内部创建了名为 `.venv` 的 Python 虚拟环境，原环境内的绝对路径引用（如 pip 的 wrapper、activate 脚本等）均已失效。请按照以下步骤重建虚拟环境：

```powershell
# 1. 进入更名后的项目根目录
cd D:\AIZS

# 2. 物理删除已失效的虚拟环境目录
Remove-Item -Recurse -Force .venv   # Windows PowerShell 
# rm -rf .venv                      # MacOS/Linux/Git Bash

# 3. 创建全新的 Python 3.11 虚拟环境
python -m venv .venv

# 4. 激活虚拟环境
.venv\Scripts\Activate.ps1          # Windows PowerShell
# source .venv/bin/activate         # Linux/MacOS

# 5. 重新安装所需的全部依赖包（含新加入的 python-docx 依赖）
pip install -r requirements.txt
```

### 3. 配置文件修改

复制 `.env.example` 为 `.env` 并填写相关信息。确保已填入有效的 `DASHSCOPE_API_KEY` 字段：

```bash
cp .env.example .env
```

建议在答辩或评估中以如下流程展示项目：

   - 点击“进入管理端”。在登录框输入 `admin`/`admin123` 登录。
   - 在定制侧边栏点击 **📚 知识库导入与管理**，选择「演示文档一键导入」標签，点击一键导入演示知识库，观察页面进度条及处理状态（显示文件名、切片数等信息）。

## 🛑 常见问题与排查指南

### 1. 启动 Streamlit 提示 `ModuleNotFoundError`
- **原因**：由于重命名目录，原虚拟环境失效；或者未正确激活虚拟环境。
- **解决方法**：请删除根目录下的 `.venv` 并根据 [环境配置与安装](#️-环境配置与安装) 部分的“虚拟环境重建”说明重新创建并拉取依赖。

### 2. 意图匹配或聊天无法给出有效回答
- **原因**：大模型调用被拒绝，可能是 `.env` 中的 `DASHSCOPE_API_KEY` 填错或过期。
- **排查方法**：进入管理端，点击侧边栏"🩺 系统自检"，执行一键自检查看百炼 API 测试项，检查是否发生 `401 Unauthorized` 错误。
